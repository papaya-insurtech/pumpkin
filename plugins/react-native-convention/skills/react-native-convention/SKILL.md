# React Native Convention

Apply these conventions when writing or reviewing React Native code in Papaya mobile projects.

## Tech Stack

- [React Native](https://reactnative.dev/)
- [Expo](https://expo.io/)
- [Apollo Client](https://www.apollographql.com/docs/react/)
- [gql.tada](https://gql-tada.0no.co/)
- [TypeScript](https://www.typescriptlang.org/)
- [React Native Paper](https://callstack.github.io/react-native-paper/)

## General Rules

- Follow the `typescript-convention` skill for all TypeScript code.
- Follow the `react-convention` skill for all React patterns (conditional rendering, hooks, etc.).
- Use React Native Paper components as the primary UI library.
- Use Expo APIs and modules wherever available instead of bare React Native or third-party alternatives.
